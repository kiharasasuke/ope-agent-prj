# Airペイ手数料計算スキル

## 概要

Airペイの決済明細CSVから、入金サイクル別・決済ブランド別に手数料を集計計算する。

## 手数料率

詳細は [references/fee_rates.md](references/fee_rates.md) を参照。

### キャンペーン適用時（2.48%）
- Visa, Mastercard, JCB, American Express, Diners Club, Discover

### 通常料率
- iD, QUICPay: 3.24%（非課税）
- 交通系電子マネー: 2.95% + 税 = 3.245%（課税）
- QR決済（PayPay, d払い, 楽天ペイ等）: 2.95% + 税 = 3.245%（課税）

## 入金サイクル

Airペイは月3回入金:
- 10日〜19日の売上 → 月末入金
- 20日〜月末の売上 → 翌月10日入金
- 1日〜9日の売上 → 翌月17日頃入金

## 使用方法

### 1. CSVファイルの確認

Airペイ管理画面からダウンロードした「全項目」CSVを使用。
CSVの構造:
- 列0: 入金日
- 列5: 決済ブランド
- 列8: 金額

### 2. スクリプト実行

```bash
python scripts/calculate_fee.py <input_csv> [output_csv]
```

引数:
- `input_csv`: Airペイの決済明細CSV（必須）
- `output_csv`: 出力先CSV（省略時は `手数料集計_YYYYMM.csv`）

### 3. 出力形式

CSV出力:
```
入金日,決済ブランド,合計金額,課税区分,手数料率,手数料
```

## 月末日分離機能

月末日～翌月9日の入金サイクルでは、月末日の売上を自動的に分離して集計。
会計期間を正確に分けるための機能。

例: 11/17入金分
- 10/31（月末）の売上 → 10月分として分離
- 11/1〜11/9の売上 → 11月分として集計

出力CSVには会計月別サマリーも含まれる。

## 弥生仕訳生成

### スクリプト実行

```bash
python scripts/generate_journal.py <input_csv> <department> [subsidiary_suffix] [output_csv]
```

引数:
- `input_csv`: Airペイ全項目CSV（必須）
- `department`: 弥生部門名（例: 大衆サカバ牡蠣ル）
- `subsidiary_suffix`: 売掛金補助科目の店舗略称（省略時は部門名）
- `output_csv`: 出力ファイル（省略時は `仕訳_YYYYMM.csv`）

### 仕訳形式（4行1組）

| 行 | 借方科目 | 借方補助 | 借方税区分 | 貸方科目 | 貸方税区分 |
|----|---------|---------|-----------|---------|-----------|
| 1 | - | - | 対象外 | 売上高(10%) | 課税売上込10% |
| 2 | クレジット手数料 | (課税) | 課対仕入込10%適格 | - | 対象外 |
| 3 | クレジット手数料 | - | 非課仕入 | - | 対象外 |
| 4 | 売掛金 | ㈱ﾘｸﾙｰﾄﾍﾟｲﾒﾝﾄ/店舗 | 対象外 | - | 対象外 |

### 部門一覧

- 事業所 / 本部 / 赤坂元気(仮)
- Crab Shrimp and Oyster
- 牡蠣と和牛の奴隷
- 大衆サカバ牡蠣ル
- スタンドバイミート
- カキノバル

## 注意事項

- CSVが重複データを含む場合、自動検出して有効行のみ処理
- 手数料は小数点以下切り捨て
- 返金（マイナス金額）は手数料もマイナスで計算
- 月末日は自動判定（28日/29日/30日/31日）
- 課税手数料は既に税込金額（消費税を二重計上しない）
